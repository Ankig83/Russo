# РУССО — Сайт мебельной компании

## О проекте

Сайт мебельной компании «Руссо» с интерактивным 3D шкафом как главным навигационным элементом.
На главной странице стоит реальный шкаф компании в интерьере комнаты. Пользователь открывает
дверцы шкафа — внутри появляются ящики, каждый из которых является переходом в раздел сайта.

## Структура шкафа (навигация)

```
┌─────────────────────┐
│   [ЗАКРЫТЫЙ ШКАФ]   │  ← стартовое состояние
└─────────────────────┘
          ↓ клик
┌──────────┬──────────┐
│  ◄ влево │ вправо ► │  ← дверцы распахиваются (door_left, door_right)
├──────────┴──────────┤
│     [ drawer_1 ]    │  ← центральный верхний (большой)
├──────────┬──────────┤
│ drawer_2 │ drawer_3 │  ← коллекция ГАФТ
├──────────┼──────────┤
│ drawer_4 │ drawer_5 │  ← коллекция КЕФА
└──────────┴──────────┘
```

### Объекты в GLB файле
| Имя объекта | Тип анимации | Раздел сайта |
|---|---|---|
| `korpus` | статичный | — |
| `door_left` | поворот по Y влево | раздел (уточнить) |
| `door_right` | поворот по Y вправо | раздел (уточнить) |
| `drawer_1` | выдвижение по Z | раздел (уточнить) |
| `drawer_2` | выдвижение по Z | ГАФТ левый |
| `drawer_3` | выдвижение по Z | ГАФТ правый |
| `drawer_4` | выдвижение по Z | КЕФА левый |
| `drawer_5` | выдвижение по Z | КЕФА правый |
| `details` | статичный | ножки, ручки |

## Архитектурные решения

**Почему GLB а не OBJ** — GLB один файл с текстурами внутри, быстрее грузится, нативно поддерживается Three.js.

**Почему Zustand а не Redux** — проще, меньше бойлерплейта, достаточно для корзины и авторизации.

**Почему FastAPI а не Node.js** — разработчик знает Python, FastAPI современный и быстрый, автодокументация из коробки.

**Почему React Admin** — не писать админку с нуля, экономия 2-3 недель работы.

**Масштабируемость шкафа** — разделы хранятся в массиве `sections`, добавление нового раздела = одна строка в конфиге без изменения компонентов.
